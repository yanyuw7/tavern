# plot 层条目写法（演绎指南 / 文风指引 / 可执行条目）

给"剧情 AI"看的层。**职责边界**：只写"是什么/怎么演/怎么写"；出现字段名、路径、JSONPatch、schema 清单 = 立刻撤走归 update 层（额外解析模式下剧情 AI 根本收不到变量规则，写了也白写还占 token）。

## 1. 数值演绎指南（数值 → 行为）

把 stat 数值翻译成演出，不让 AI 在正文里报数。骨架：

```
核心原则（放条目最前）:
1 不数值化叙事——永不写"她的羞耻度只剩20"
2 不照抄本指南任何句子——只描述维度，具体描写按场景自由生成
3 数值决定写什么，不是写什么数字
4 每个角色独立判断——同一事件不同人反应不同
5 避免人均极端——30~70 是常态，90+ 是少数
6 数值变化渐进——单次普通事件小幅，重大事件中幅

每个数值一节:
  分档表（0~10 / 10~30 / ... / 90~100 各一句"状态描述"）
  演绎维度清单（身体侧/心理侧/语言侧——只给维度名，不给例句）
  ⚠️ 反例 1-2 条（"每次都夹紧大腿=单一动作复读"）
  与相邻数值的边界（例：领域专属的"羞耻"≠日常社交害羞——归性格维度管；"敢不敢做"和"敢不敢被看见"是两个正交维度，别合并成一个数值）
```

**防复读三禁**：禁可套用例句（LLM 逐字照抄）；禁"规定动作/标志台词"（好感阶段写死动作 = 全员机器人）；性格演绎只留一句方向（"据 Tags 演绎本角色的性格与说话方式"），"反照抄纪律"这类元提示没用，删。

**SFW/NSFW 拆分**：数值按生效面分两组——**长期人格类**（价值观偏移/负罪感/自制力这类日常也起作用的）写常驻条目、只给日常情境维度、完全不出现身体描写；**场景状态类**（仅亲密场景有意义的兴奋/敏感类）单独条目挂 `@@if`（表达式判"相关数值超阈值或相关经历计数>0"）——常驻发 NSFW 示例会把 SFW 阶段也带偏。

## 2. 文风指引（正文质感锚点）

**存在的根本理由**：LLM 没有稳定文风锚点时，会把"刚才的笔触/具体动作"回填进变量字段"备忘"（实录：一个偏好类列表被塞成 38 条事件流水）。有了指引，LLM 知道下一轮也能复现质感，变量才守得住"只存状态"。所以指引开头就写"与变量系统的分工"表。

骨架（实战验证过）：

```
0 总基调（一句话气质定位 + 要/不要对照表 + "写完问自己：这段有意思吗"）
1 叙事视角（第三人称聚焦 + 视角物理锁定：角色无读心无全知，只能靠表面信息+主观误解）
2 描写焦点：外化优先（视线轨迹/姿态/空间/材质光感/声音/旁观者反应；禁直陈心理）
3 句式节奏（形容词≤2/句；禁排比、四字辞藻连排、抒情修饰）
4 对话（允许长句连说像活人聊天；标签默认"说"；禁 ABAB 短句循环/采访式问答）
5 心理处理（独白节流：仅强冲击/重大决策/掩饰欲望三情形开启；格式【角色名内心：≤2句】；
   反元数据泄露：机制词"食髓知味/底层逻辑/触发阈值"禁进正文；反临床自我分析）
6 群像调度（多核心角色像切镜头流转；旁观者 1-3 人反应各异不写心理）
7 细节锚点（每段 1-2 个物理锚点：衣物部位/光源/材质/声音/温度气味；塞 5 个=细节流水账）
8 禁忌清单（文体八股：AI 腔说明文/总分总/言情体/老干部体/网文模板腔/万能开头结尾；
   临床数据：禁 cm/秒/百分比→改相对感知；禁无主语电报体；
   悬而未决：段尾禁反问/省略号水字/等待玩家许可——动作必须落地）
9 节奏（高强度后必有缓冲段；时间推进用动作显示）
10 NPC 决断阈值（犹豫≤1回合，下轮必须结算；禁反问把决策抛回玩家）
11 写完自检（有意思吗/八股扫描/对话扫描/+回填变量时是否在塞笔触→删）
```

NSFW 专项另附（挂在亲密指南尾部）：肢体多线程（双手/嘴/视线不空闲）、阶梯快感累积反秒潮、事后强装镇定必须有破绽、语言降级但过角色人格滤镜（禁全员同款淫语）。

反 AI 腔病灶四大件：完整流畅句式（真人说话有毛刺）/ 每句配动作 / 角色解释自己 / 全员统一语速。

## 3. 可执行条目体例（情景/玩法条目）

服装/露出/玩法类条目的三段式（比纯描述性条目执行率高得多）：

```
## 触发条件（什么场景激活）
## 执行步骤（编号动作清单，"每次至少选 N 项"）
## 示例（对比）
- ❌ 错误:"她穿着一件露出服装。"（概括帽子）
- ✅ 正确: 完整正例一段
```

多源融合先划职责边界（"A 书管怎么叫，B 书管怎么讲故事"），条目里显式写 `[接口]` 段声明分工，防条目互相打架。

## 4. 选择栏（每轮末尾选项）

常驻 + order 压轴（9999）。格式 `编号|类型标签|选项内容`。**做卡前必过访谈**（SKILL.md 第二轮必问项）：要不要、几个、类型标签集、文案规范选哪派、要不要美化——这些全是作者偏好，不替作者定。

**文案规范两派**（访谈的核心分歧点）：

| | 可直发派 | 氛围派 |
|---|---|---|
| 规范 | **强制完整主谓宾，主语必须是 {{user}}**（"{{user}}推开生锈的铁门"） | 允许短语/疑问/开放式（"推开铁门""要相信她吗？"） |
| 原理 | 选项被玩家原样发送 = 一条清晰的玩家行动输入，AI 零歧义 | 更像游戏 UI 的选项气质，短而有味 |
| 代价 | 句子长、每轮 token 多、文案略呆板 | 发送后主语缺失——AI 可能把动作按给错的人、或把疑问句当玩家台词 |
| 适配 | 多角色在场的卡（主语歧义高危）、变量卡（行动要被变量 AI 正确归因） | 单主角强、正则美化成按钮点击发送的卡（点击语义已经=玩家选择） |
| 补救 | — | 氛围派若配美化按钮，可在正则替换时给发送文本自动补 "{{user}}选择了：" 前缀，两头兼顾 |

其余设计参数（也在访谈定）：选项数（4~10 常见，多=引导性强/token 贵）；类型标签集按卡的玩法定（推进/互动/转折/日常/NSFW/系统），标签本身要求 AI 每轮换着出防单一化；末尾可留一个"自由行动"位提示玩家可以不选。
配套：美化正则 + 消除兜底（regex.md §2.B）；类型标签集写死在条目里（枚举），AI 自创标签会让按类型染色的美化正则失色。

## 5. plot 层 token 纪律

- 常驻 plot 条目控制在各 ~3k 字符内；指南类超了就拆 @@if / key 触发。
- 演绎指南只写"维度+反例"不写示范段落——示范段落 = 复读源 + token 大户。
- 世界观设定按题材拆 key 触发条目（组织/功法/战斗各自触发），不塞主规则。
- 特定风格词会让玩家出戏的（"几丁质/外骨骼"这类硬词）全文清除。
